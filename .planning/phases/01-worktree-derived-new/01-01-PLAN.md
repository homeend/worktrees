<design_constraints>
<!-- Locked decisions and verified facts. Honor exactly. -->

- DERIVE-MODE TRIGGER PREDICATE (gate for DETECT-01/02 + regression safety): activate derive mode ONLY when ALL hold:
  (a) the listed worktree containing `dir` is NOT the main worktree (its Path != MainRoot(dir)), AND
  (b) the invocation supplies no explicit branch-source: opts.Branch == "" AND opts.FromBranch == "" (template resolves into opts.Name, so a non-empty opts.Name from --template also disqualifies — see below).
  If any condition fails → fall through to TODAY'S behavior unchanged (DETECT-02). Detection uses the existing ListWorktrees; pick the entry whose Path is the longest path-separator-bounded prefix of `dir` (cwd may be a subdir of the worktree). The container is a SIBLING of repoRoot, so no false ancestor match against main.
- DERIVE-01 (committed tip): set baseRef to the parent branch NAME (e.g. `wt/feature-login`). `git worktree add -b <new> <path> <parentBranch>` cuts from that ref's commit — uncommitted changes are excluded by construction. No copy machinery.
- DERIVE-02 (placement): keep targeting MainRoot(dir) and reuse worktreePath unchanged. New worktree lands in the main container, mirroring the full branch path. Do NOT nest under the current worktree.
- DERIVE-03 (prefix verbatim): in derive mode the new branch = parentBranch + suffix; parentBranch already carries its prefix. Do NOT call effectivePrefix / apply NoPrefix / PrefixOverride / cfg.BranchPrefix() in derive mode. Also: `--base` is IGNORED in derive mode (baseRef is the parent branch).
- NAME-01 / NAME-02 (auto suffix): when token is empty, branch := first free `fmt.Sprintf("%s-v%03d", parentBranch, n)` for n starting at 1, increment while BranchExists(repoRoot, candidate) — lowest free, gap-filling, zero-padded width 3.
- NAME-03 (custom token): token is the literal positional arg (e.g. `-patch01`). branch := parentBranch + token (appended verbatim, NO separator inserted — the user's leading dash is the separator). This REPLACES auto-numbering.
- NAME-04 (custom collision = hard error): if the custom-token branch already exists, return a clear error (e.g. `branch %q already exists` naming the derived branch) — NO rename, NO auto-bump. This is a SEPARATE code path from today's generic "pass a different --branch" message and from the auto-vNNN skip loop.
- TOKEN PLUMBING (verified): cobra flag-parses a bare leading-dash arg (`wt new -patch01` → "unknown shorthand flag: 'p'"). The token reaches args intact via the `--` separator: `wt new -- -patch01` → args[0] == "-patch01". Keep CLI dumb: positional → opts.Name (existing behavior); `Manager.Add` reinterprets opts.Name as the literal suffix token ONLY when derive mode is active (empty opts.Name in derive mode → auto -vNNN). Do NOT add a dedicated Suffix field. Document the `--` requirement in the command help/long text.
- CONSTRAINTS (CLAUDE.md): extend Manager.Add / AddOptions / GitRunner — do not bypass; no main-root regression; unit-testable via injected fakes (no real git in unit tests); derived names pass CheckRefFormat + collision checks before creation.
</design_constraints>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → generated git ref + filesystem path | The custom suffix token and the derived parent branch name flow into a new branch ref AND (via worktreePath) into an on-disk directory path under the container. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Custom suffix token → derived branch ref (manager.go Add) | mitigate | The derived branch passes the existing `m.git.CheckRefFormat(branch)` gate before any creation; an illegal token (e.g. embedded `..`, control chars, leading/trailing junk) is rejected. Keep this gate on the derive path. |
| T-01-02 | Tampering | Derived branch → worktreePath directory (path traversal) | mitigate | worktreePath mirrors the branch under the container via filepath.FromSlash; CheckRefFormat already forbids the ref forms (`..`, leading `/`, etc.) that would escape the container, so no traversal is reachable from a ref that passed the gate. No separate path sanitization needed. |
| T-01-03 | Spoofing/Detection error | Worktree-vs-main detection (currentWorktreeBranch) | accept | Detection compares the listed worktree Path against MainRoot; container is a sibling of repoRoot so a false main↔worktree match is not reachable. Local single-user CLI, low-value target — residual risk accepted. |
| T-01-SC | Tampering | Dependency installs | accept | No new third-party dependencies are added in this phase (Go stdlib + existing cobra only); no package-legitimacy gate required. |

Risk profile is proportionately LOW: a local single-user CLI shelling out to the user's own git, no network, no new dependencies. The single load-bearing control is the existing `CheckRefFormat` gate on the derived branch.
</threat_model>
